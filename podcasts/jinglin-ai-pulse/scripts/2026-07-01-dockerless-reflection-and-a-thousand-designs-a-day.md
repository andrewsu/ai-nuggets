# Ollie's AI Pulse — Dockerless, Reflection, and a thousand designs a day

## Sources

- **Dockerless: Environment-Free Program Verifier for Coding Agents** — arXiv 2606.28436, Wenhao Zeng + Yuling Shi + Xiaodong Gu + Chao Hu + Chaofan Wang + Yuhao Cui + Hongting Zhou + Mengnan Qi + Jianqiao Wangni + Zhaojian Yu + Shuzheng Gao + Kai Cai + Shilin He (Shanghai Jiao Tong University + ByteDance), HF Daily Papers July 1, 75 upvotes. A verification system for coding agents that evaluates generated patches without executing them. Instead of Docker-based reference-solution comparison, the verifier gathers evidence through agentic repository exploration — reading the repo the same way a senior engineer would. Headline benchmarks — 62.0% on SWE-bench Verified, 50.0% on SWE-bench Multilingual, 35.2% on SWE-bench Pro; 14.3 AUC points above the strongest open-source verifier. Beats a Qwen3.5-9B baseline by 2.4 / 8.7 / 2.9 points across the three SWE-bench splits. The verifier is used in both supervised fine-tuning and reinforcement learning stages of coding-agent training — the same signal that judges the final patch supervises the learning loop.
- **DOPD: Dual On-policy Distillation** — arXiv 2606.30626, Xinlei Yu + Gen Li + Qingyi Si + Guibin Zhang + Yuqi Xu + Congcong Wang + Shuai Dong + Kaiwen Tuo + Xiangyu Zeng + Kaituo Feng + Qunzhong Wang + Yang Shi + Xiaobin Hu + Xiangyu Yue + Jiaqi Wang + Shuicheng Yan (16 authors), HF Daily Papers July 1, 57 upvotes. Advantage-aware dual distillation that dynamically routes token-level supervision between a privileged teacher policy and a privileged student policy based on the advantage gap between them and their relative probabilities. Names a specific failure mode — the "privilege illusion" — where naive on-policy distillation conflates the transferable capability gap the student is meant to close with the information-asymmetry gap that can only be mimicked, not replicated. Consistently outperforms vanilla OPD across LLM and VLM settings on stability, robustness, continual learning, and out-of-distribution tasks.
- **BlockPilot: Instance-Adaptive Policy Learning for Diffusion-based Speculative Decoding** — arXiv 2606.31315, Hao Zhang + Yiming Hu + Yong Wang + Mingqiao Mo + Xin Xiao + Xiangxiang Chu, HF Daily Papers July 1, 60 upvotes. Argues that fixed block sizes across all inputs are suboptimal for diffusion-based speculative decoding; the ideal block size varies per sample. BlockPilot predicts optimal block size from the prefilling representation — one prediction per input. Result — 4.20× speedup on Qwen3-4B with acceptance length 5.92 at temperature T=1; plug-and-play with minimal compute overhead. Leverages the observation that optimal block sizes exhibit clear local structure, so the decision space is low-dimensional.

## Script

Good morning, Ollie. This is Ollie's AI Pulse for Wednesday, July first.

Today the shape of the day is the same shape the show has been tracking for a week — the agent, not the model — but the specific move is new. Yesterday it was horizon scaling — extend the trajectory, keep the parameters. Today it's verification without execution — drop the sandbox, read the repo. And on the money layer, today is the day the payments start on a six-billion-dollar compute contract that puts the second American open-weight frontier lab inside Elon Musk's data center. Let's go.

First — Twitter Pulse.

The lead paper on Hugging Face Daily Papers this morning is a ByteDance and Shanghai Jiao Tong University collaboration, thirteen authors led by Wenhao Zeng and Yuling Shi. Title — quote — "Dockerless: Environment-Free Program Verifier for Coding Agents." Seventy-five upvotes overnight, second on the daily list only because Orca is still coasting on its Tuesday momentum. The claim is unusually direct. When a coding agent proposes a patch, the standard way to check the patch is to spin up a Docker container, install the repo's dependencies, run the test suite, and score the result. Dockerless refuses to do that. Instead, the verifier is itself an agent. It reads the repository — the source code, the file layout, the imports, the tests, the commit history — and judges whether the patch is correct from evidence rather than execution. Headline benchmarks — sixty-two percent resolve on SWE-bench Verified, fifty percent on SWE-bench Multilingual, thirty-five-point-two percent on SWE-bench Pro. Beats a Qwen three-point-five nine-billion-parameter baseline by two-point-four, eight-point-seven, and two-point-nine points across the three splits. Fourteen-point-three A-U-C points above the strongest open-source verifier. And crucially, the same signal that judges the final patch also supervises training — the verifier plugs into both supervised fine-tuning and reinforcement learning during coding-agent training. The signal is the same signal.

Read that against yesterday. Yesterday's argument was horizon scaling — the model's own multi-tens-of-thousands-of-tokens trajectory replaces trillion-parameter capacity. Today's argument is the same argument applied to verification. The Docker sandbox is a heavyweight execution loop. The Dockerless verifier is a lightweight agentic exploration loop. Both papers say — the code path the agent takes through the problem is where the intelligence lives. If the trajectory is long enough and structured enough, it doesn't matter whether the code actually ran, or whether the model has a trillion parameters.

Third and fourth papers landed the same day. DOPD — Dual On-Policy Distillation — from a sixteen-author team, fifty-seven upvotes, continues last week's on-policy distillation cluster. The move is specific. On-policy distillation gives the student model a strong teacher signal, but naive OPD conflates two very different kinds of gap — a capability gap the student can actually close, and an information-asymmetry gap the student can only mimic. The paper names this the — quote — "privilege illusion." DOPD's answer is advantage-aware routing at the token level. For each generated token, the system checks which policy — teacher or student — has the higher advantage in this specific context, and routes supervision accordingly. Different token, different teacher. Fifth paper — BlockPilot from a six-author team, sixty upvotes — attacks the same instance-adaptive question on the inference side. Diffusion-based speculative decoding uses a fixed block size across every input. Turns out the optimal block size is per-sample, and predictable from the prefilling representation alone. BlockPilot predicts it in one shot and gets a four-point-two-times speedup on Qwen three four-billion at unit temperature. Same shape as DOPD — one decision per token, or one decision per prompt, but adapt.

If you're keeping score of this week's meta-argument — the field has stopped scaling one thing and started routing at another. Yesterday, route the horizon. Today, route the verification. Route the token. Route the block. The agent is the unit of progress, and the agent is a set of adaptive routing decisions wrapped around a model that is, increasingly, held constant.

The biological translation is direct and slightly uncomfortable. The Dockerless recipe — that the verifier reads the repository the same way a senior engineer would, rather than running the code — is the same recipe your field wants for wet-lab verification. Not — run the experiment, wait three months, see the readout, feed the readout back to the model. But — the model reads the experimental plan the way a senior biologist would, and predicts whether it will work from evidence rather than execution. That's the missing scaffold between a cell-foundation model and a real experimental loop, and it's being built out in the coding-agent domain right now with a benchmark score already published. Whoever ports Dockerless into perturbation biology wins a real lead.

That's Twitter Pulse.

Next — Money Moves.

Today is the day the payments start. Ten days ago — Monday June twenty-second — Axios broke the story, and CNBC, TechCrunch, Forbes, and Bloomberg followed inside twelve hours. SpaceX signed a compute agreement with Reflection AI. Terms — one hundred fifty million dollars per month, effective July first — that's today — running through twenty twenty-nine. Total value up to six-point-three billion dollars over the contract. Either side can walk away with ninety days' notice after the first three months. Compute is Nvidia G-B three-hundred systems inside SpaceX's Colossus two data center near Memphis, Tennessee.

Two things about this deal are what makes it today's story rather than last week's.

First — who is Reflection A-I. Founded twenty twenty-four by two former Google DeepMind researchers — Misha Laskin and Ioannis Antonoglou. Antonoglou was on the AlphaGo team and later on Gemini. This is the same DeepMind alumni-diaspora pattern we tracked all last week — Shazeer to OpenAI, Jumper to Anthropic, Adler and Pritzel to Anthropic. Laskin and Antonoglou left DeepMind eighteen months earlier and instead of joining an established lab they built one. And Reflection isn't a scaling clone — its stated positioning is — quote — "the DeepSeek of the West" — an American open-weight frontier lab. Their product Asimov, launched last summer, is a code-comprehension agent that reads a company's codebase, architecture docs, GitHub threads, chat history, emails, and Slack, and builds persistent memory of the whole engineering organization. Sequoia's write-up says Asimov's answers were preferred a majority of the time in blind tests over Cursor Ask and Claude Code — Sonnet three-point-seven and four — by maintainers of some of the largest open-source projects. Reflection was funded at eight billion post-money in October twenty-twenty-five — two billion dollars from Nvidia, Eric Schmidt, Citigroup, Lightspeed, Sequoia. In May they signed a partnership with the Department of Energy's Genesis Mission. Today, they take delivery of the compute.

Second — where the compute lives. As of this morning, the map of American frontier-scale compute is essentially — Colossus one and Colossus two. Anthropic is renting all of Colossus one at one-point-two-five billion dollars per month — two hundred twenty thousand Nvidia G-P-Us, three hundred megawatts of power, roughly forty-five billion dollars through mid-twenty twenty-nine. Google is renting Colossus two capacity at nine hundred twenty million dollars per month — about thirty billion dollars through the same window. Reflection just booked a smaller slice of Colossus two at one hundred fifty million per month — six-point-three billion through twenty twenty-nine. And SpaceX bought Cursor outright two weeks ago in a sixty-billion-dollar all-stock deal. The physical substrate under American frontier A-I — Anthropic, Google, and the second-largest open-weight American lab, plus the fastest-growing coding editor — all runs, or is bought by, a data-center operator owned by Elon Musk. The frontier-lab layer above and the compute-supply layer below have just re-integrated under one ownership node.

Read that against last week's Anjney Midha take on Latent Space. Midha's thesis was that the substrate is the bottleneck and the frontier-lab game is going to be run as an Independent System Operator — pool multi-cloud demand against multi-silicon supply, make F-L-O-P-S flow like megawatts. Colossus is running the exact opposite play. Instead of pooling supply across many operators, it's concentrating supply under one operator and letting different frontier labs bid for slices of it. Reflection didn't get the ISO deal. It got a landlord.

The Ollie-specific read on this. If you are a biological-foundation-model group with a big Perturb-seq or a big multi-modal-cellular training set to do this year, and your compute plan is "we'll figure out the cloud spot market when we get closer," look carefully at these numbers. One-point-two-five billion a month. Nine hundred twenty million a month. One hundred fifty million a month. Not for training run — for standing capacity, month over month, through twenty twenty-nine. The frontier text labs are consuming compute at a per-month rate that would fund fifty biological-foundation-model groups a month. The Perturb-seq consortia don't have a Colossus. The cell-atlas groups don't have a Colossus. The AlphaFold-of-the-cell contender is going to have to either share a slice of the frontier-text stack or persuade a national lab to build the biological analog. That's a decision that needs to be made this year, not next.

That's Money Moves.

Now — Podcast Deep Cut.

Nathan Labenz on The Cognitive Revolution, published yesterday June thirtieth, ninety minutes. Guest — Thomas von Tschammer, co-founder and U-S managing director of Neural Concept, an E-P-F-L spinout from twenty nineteen that just closed a hundred-million-dollar Series C from Goldman Sachs in December. Title — quote — "1000 Designs a Day: Neural Concept's Thomas von Tschammer on A-I-Native Engineering."

His argument is the AI-native engineering argument, and it's the closest analog outside of biology to what your field needs to sound like a year from now.

Von Tschammer's frame is that engineering has had three revolutions. First, forty years ago — physical prototypes gave way to numerical simulation. Second — numerical solvers wrapped in C-A-D and finite-element-analysis software let engineers evaluate fifty to a hundred designs per year. Third — physics-aware A-I lets them evaluate thousands per day. His direct quote on the mechanism — quote — "Thanks to A-I, you don't get results in days, but minutes. If you can get results in minutes, you explore thousands of options."

The load-bearing case study is Jaguar Land Rover. Aerodynamic design evaluations in production went from fifty per day to one thousand five hundred per day. Von Tschammer's phrasing — quote — "They went from fifty designs evaluated per day to one thousand five hundred every single day in production." Battery-cooling supplier case — eighty percent shorter development cycles, twenty percent better cooling, fifteen percent lighter batteries. Formula One — the sport already runs a sliding-scale C-P-U cap on aerodynamic simulation, where better finishers get fewer compute hours the following season. The governance precedent is already there — simulation budget is a competitive lever, and it's already being regulated.

The strategic frame is worth extracting. Von Tschammer says foundation models for aerodynamics are — quote — "low-hanging fruit" and are coming soon. Neural Concept's move is deliberately not to compete as first-mover on the foundation-model layer. Their strategy is to be the domain-specific integration layer — because plain L-L-Ms, in his words, "lack very accurate 3-D reasoning" and cannot solve fluid dynamics. His deeper argument is that the engineering I-P a company embeds into its A-I workflows is the durable differentiator. Quote — "The ones that deeply ingrain their engineering I-P into these A-I workflows will differentiate tomorrow." And on the competitive stakes he is stark — quote — "Competing with Japanese firms in the seventies and eighties may be easy mode compared to competing with A-I-native companies." Chinese O-E-Ms already run eighteen-to-twenty-four-month development cycles against Western O-E-Ms' forty-eight-to-sixty-month cycles. Neural Concept's two-year adoption roadmap for legacy O-E-Ms — Year One, A-I-led iteration inside crash, aero, and powertrain silos gives twenty to forty percent cycle speedups. Year Two, cross-disciplinary orchestration across those silos gives fifty to sixty percent cycle reduction.

And one specific line worth marking — von Tschammer's Move-Thirty-Seven framing — quote — "Engineers thought it was a blunder, but it actually worked. Models escape bounds of human intuition while grounded by physics." The physics constraint is what lets the model diverge safely from human intuition. Without physics, you get hallucination. With physics, you get Move Thirty-Seven.

The Ollie-specific take is the tight one. Substitute "cell" for "aerodynamic" in every one of von Tschammer's sentences. "One thousand five hundred Perturb-seq designs evaluated per day in production." "The teams that deeply ingrain their biological I-P into these A-I workflows will differentiate tomorrow." "Cell foundation models are low-hanging fruit and are coming soon; the moat is the domain-specific integration layer." "Models escape bounds of human intuition while grounded by biological measurement." The playbook is the same playbook, and it works when three conditions are met — first, the model is trained on a mixture of simulated data and real measurement rather than either alone. Second, the domain-specific integration layer is where the I-P sits — not in the base weights. Third, the physics — or in your case, the biology — is the constraint that lets the model safely surprise a human. If your group's cell-foundation-model plus Perturb-seq loop doesn't look like Neural Concept's platform two years from now, some other group's will, and that group will be the one running fifteen hundred experiments a day while the field is still doing fifty.

That's the deep cut.

Wrap.

Watch one thing this week — whether any biological A-I group publicly frames its next cell-foundation-model plus experimental-loop release as — quote — "one thousand Perturb-seq screens per day," or the equivalent, in the JLR frame Neural Concept just took to Nathan Labenz. The naming matters. Neural Concept has been running a hundred-million-dollar Series C off the "one thousand designs a day" line for six months. There is no equivalent line in cell biology. First group to plant that flag on a real production number wins the narrative for the AlphaFold-of-the-cell race. Have a good Wednesday, Ollie.
