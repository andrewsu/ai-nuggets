# AblateCell — An Agent That Reproduces and Ablates Virtual Cell Models in Their Native Repos

Paper link: https://arxiv.org/abs/2604.19606

## Summary

A team led by Xue Xia just dropped AblateCell, a two-phase LLM agent that takes a virtual cell repository — CPA, GEARS, BioLORD — and reproduces the reported baseline end-to-end, then runs principled ablations on its own. End-to-end reproduction success: 88.9%, which is 29.9 points above the human expert baseline they benchmarked against. Critical-component identification: 93.3%, which is 53.3 points above heuristic ablation. The reason this matters isn't the headline numbers — it's that virtual cell papers are piling up faster than anyone can verify them, and the only honest way to attribute a performance gain is the boring, painful work of standing the model up and knocking pieces out. AblateCell is the first system that does that without a postdoc burning two weeks per repo.

## Script

AblateCell. If you've ever tried to clone a virtual cell model repo from the supplementary of a flashy preprint, and then actually reproduce the number in the abstract, you know that the gap between "the code is on GitHub" and "I can rerun the headline result" is enormous. There are missing dependencies. There are paths hardcoded to someone's HPC cluster. There are config flags that were renamed between paper submission and code release. There are data files behind a Google Drive link that returns a 403. The reproducibility problem in AI for biology is not theoretical — it's a friction tax that gets paid every time anyone tries to build on someone else's foundation model.

A team led by Xue Xia, with collaborators including Cheng Tan, Siqi Sun, and Zhangyang Gao, just put up a paper on arXiv called AblateCell — a reproduce-then-ablate agent for virtual cell repositories. Version one went up April 21st. Version two landed April 30th. And the framing is exactly what the field has been waiting for someone to build.

The pitch is two phases. Phase one: the agent takes a public virtual cell repository — they tested on CPA, GEARS, and BioLORD, all single-cell perturbation prediction models — and reproduces the reported baseline end-to-end. That means it auto-configures the conda environment, resolves the dependency conflicts, fetches the data, fixes the broken paths, and reruns the official evaluation script until the number matches. Phase two: it does ablations. It generates isolated mutations of the model — drop this attention head, swap that loss term, replace this pretrained encoder with a baseline — and strategically picks which ablations to actually run, because compute is finite. The end product is an attribution map: for each component the original authors claimed was important, AblateCell tells you whether knocking it out actually hurts performance.

The numbers they report are striking. End-to-end workflow success rate: 88.9 percent. That's the percent of the time AblateCell can take a virtual cell repo it has never seen and produce the reported baseline result without human intervention. The human expert baseline they benchmarked against came in 29.9 points lower. Let that sink in. An LLM agent reproduces virtual cell papers more reliably than a person who actually knows what they're doing. That probably says as much about the awfulness of cell biology codebases as it does about the agent, but the practical implication is the same: the friction tax just dropped by an order of magnitude.

The other number, 93.3 percent on identifying critical components, is the one that matters more for the science. A standard ablation study, even a well-run one, often picks components to drop based on what's easy to remove, not what's actually load-bearing. AblateCell selects ablations to maximize information about which pieces are truly essential. It beats the heuristic baseline by 53.3 points. What that gives you, when you plug it into a survey of virtual cell models, is a real attribution table — not "the authors claim component X is important," but "we verified component X is important by ablating it in the original codebase under the original evaluation."

This matters for a few reasons. The virtual cell space is moving fast — Tahoe, scGenePT, scFoundation, GEARS, the list keeps growing — and most of the published gains are stacked on top of architectural choices that nobody has independently checked. Without ablation infrastructure, the field can't separate genuine contributions from incidental engineering luck. AblateCell is the first agent system explicitly built to close that loop. And unlike systems that try to do open-ended autonomous research, this one has a narrow, falsifiable job: reproduce a number, then test which pieces of the model produced it. That focus is what makes the success rates believable.

There are real limitations. Three repos isn't a benchmark, it's a pilot. The agent will need to handle the full mess of real-world virtual cell tooling — proprietary data dependencies, cluster-specific code, evaluation scripts that wrap themselves around obscure data formats. And there's a deeper question, which the authors acknowledge, about what counts as a "ground-truth critical component" — that's a label that came from somewhere, and labeling errors propagate.

But here's the bigger picture. We've spent two years watching agentic AI do increasingly impressive things on synthetic benchmarks, and increasingly fragile things on real scientific code. AblateCell is the first plausible signal that an LLM agent can land in an actual biological codebase — not a sanitized version, not a docker image someone prepared for the agent — and do something genuinely useful: verify the work. If this scales, it's a quiet but important shift. The bottleneck on building reliable foundation models for biology has always been the human cost of verification. If agents can absorb that cost, the velocity of legitimate progress in the field goes up, and the half-life of unreproducible claims goes down.

For anyone running a lab that maintains biomedical infrastructure or publishes models people are supposed to build on, AblateCell is worth watching. Not because it's done — it isn't — but because it's the right shape of thing. Reproduce first. Ablate second. Trust nothing the authors claim until the agent has confirmed it.
