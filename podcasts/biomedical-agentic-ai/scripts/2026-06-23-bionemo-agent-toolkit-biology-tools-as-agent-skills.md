# BioNeMo Agent Toolkit — OpenFold, RFdiffusion, Parabricks, and Evo2 wrapped as skills any agent can call

Paper link: https://nvidianews.nvidia.com/news/nvidia-launches-bionemo-agent-toolkit-giving-ai-agents-the-tools-to-accelerate-scientific-discovery

## Script

Today's bonus pick is the launch that landed at BIO International this morning — Nvidia's BioNeMo Agent Toolkit. The thirty-second take is that Nvidia has wrapped roughly a decade's worth of its biology toolchain — OpenFold, Boltz two, DiffDock, RFdiffusion, ProteinMPNN, Evo two, the Parabricks genomics stack — as a single, agent-callable skills marketplace, shipped open source on GitHub today. This is the moment when the "agent calls a biology tool" paradigm gets standardized. And the partner list at launch — Anthropic, OpenAI, Schrödinger, Cadence, Benchling, Databricks, Snowflake, Lilly, Natera, the Baker lab at the University of Washington, the Arc Institute, fifty-plus organizations in total — tells you why this matters more than the toolkit itself.

What did Nvidia actually ship. Four classes of components. First, NIM microservices that expose the established biology models behind a uniform agent-callable interface — OpenFold for structure prediction, Boltz two for multi-modal structure tasks, DiffDock for ligand docking, GenMol for generative chemistry, RFdiffusion for de novo protein backbone generation, ProteinMPNN for sequence design, Evo two as the genomic foundation model, and MSA Search for the multiple-sequence-alignment step that nearly every structure-prediction pipeline still needs. Second, open-model skills wrapping community efforts that are not Nvidia-native. Third, library skills for the CUDA X scientific utilities and Parabricks for accelerated alignment and variant calling. Fourth, pre-composed workflows. The one they highlight is generative protein binder design, which chains RFdiffusion to ProteinMPNN to OpenFold three as a single agent-callable skill, rather than three separate tool calls the orchestrator has to learn to stitch together.

The packaging convention is the part to read closely. Each skill ships as a directory containing a specification file in YAML, optional reference materials, and optional helper scripts. The skill format is meant to be framework-agnostic — you install through the same skills marketplace whether your agent is Claude, Codex, or any other harness that consumes the format. This is structurally important. Nvidia is not trying to own the agent layer. They are trying to own the skill layer underneath it. The agent can come from anywhere — frontier lab, internal pharma deployment, university group — as long as it speaks the skill protocol, it gets access to biology's foundational toolchain. That is a different bet than competing with Anthropic and OpenAI on agent quality, and for Nvidia's position in the stack, it is the correct bet.

The five workflow categories the press release organizes around — virtual screening, genomic analysis with Parabricks, protein binder design, deep biomedical research via what they call the Biomedical AI-Q Research Agent for literature review and protocol generation and pharmacovigilance, and medical imaging analysis — those are the canonical demand surfaces. Each one had been a custom integration project at every pharma and biotech that wanted agents in their workflow. Now it is a skills install.

Quantitative claim worth flagging, because it is the only one. Two times faster RFdiffusion three runtimes from the collaboration with the Baker lab at the University of Washington Institute for Protein Design. That is real, that is the only benchmark in the release, and the rest of the announcement is the partner list and the architecture story. Read the release as a positioning statement, not as a paper.

The Su Lab read. Four things. First, this is the inflection moment where calling biology tools from an agent stops being a research engineering project and starts being an install. The bottleneck for the past two years has been that every group writes its own orchestration layer over the same dozen foundational tools. Nvidia is collapsing that, and they are doing it by making the skill format independent of the agent. Second, look at where Anthropic and OpenAI sit on the partner list. They are the agent layer; Nvidia is the skill layer. That separation is going to define the stack for the next two or three years, and it is how you should plan your own infrastructure — assume the agent harness is interchangeable and the skills are the durable asset. Third, the obvious gap. There is no equivalent today for the knowledge-graph and ontology side of biology. You can install OpenFold and Parabricks as agent skills tomorrow; you cannot install Wikidata or the gene ontology with the same packaging discipline, the same provenance contract, the same input-output validation. That gap is exactly the opportunity for open-science groups, and it is exactly where this lab sits. Fourth, the caveat. This is a product launch, not a peer-reviewed evaluation. The skills marketplace is open source today, but the orchestration story — how agents discover, compose, and verify these skills in practice, and what happens when two skills disagree about a protein structure — is unresolved. That is the research agenda the toolkit hands the field. Talk to you tomorrow.
