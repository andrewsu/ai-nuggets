# Agent-authored single-cell embeddings — literature-cited gene programs as the compositional prior, no gene-set database, no training, no data

Paper link: https://www.biorxiv.org/content/10.64898/2026.07.07.737048v1

## Script

Today's pick inverts the last five years of single-cell embedding work. It is a preprint out of the Binder group at Freiburg, posted to bioRxiv yesterday, titled "Coding agents author interpretable single-cell embedding models from the literature."

Some setup. The dominant paradigm in single-cell analysis right now is: take an expression matrix, feed it to a neural encoder, and get back a dense latent representation you can cluster on and plot. The variants of this pattern span an enormous range. On one end you have PCA and scVI — trained per dataset, few dimensions, unsupervised. On the other end you have single-cell foundation models — scGPT, Geneformer, scFoundation, and their descendants — pretrained on tens or hundreds of millions of cells and then finetuned or probed for the task at hand. Two problems recur no matter where on that spectrum you sit. First, the resulting dimensions are dense. You cluster on them, you get clusters, and then you have to run marker-gene analysis on each cluster to figure out what the axes actually mean. Interpretation is post hoc. Second, the embeddings pick up batch effects, so you bolt on Harmony, or scVI's batch correction, or Combat, and hope for the best.

The Freiburg paper takes a genuinely different view. It says: cell biology already ships with a sparse, compositional prior. It is called the literature. Decades of experiments have identified marker gene programs — the interferon response signature, the T-cell exhaustion module, the mesenchymal program in EMT, the alveolar type two identity block — and named them, cited them, and refined them. What if instead of learning cell-state structure de novo from the expression matrix, we compile that literature directly into an embedding?

The mechanism is where the paper gets interesting. There is a structured Python template — think of it as a scaffold with slots for named axes and their component gene programs. A coding agent is given what the authors call a scenario, which is a natural-language focus on a biological subdomain. Mouse cortical development. Human tumor-infiltrating lymphocytes. Alveolar epithelium in COPD. Whatever you want the embedding to reason about. The agent then curates the relevant marker gene programs from the primary literature, cites them, and composes them into axes by editing the Python template. Crucially, there is no gene-set database in the loop — no MSigDB, no GO lookup. There is no training. There is no exposure to the actual expression data. The output is just a working embedding model, ready to be applied.

That is the core move, and it deserves a second look. This is not retrieval-augmented generation over a knowledge graph. This is the agent as an interpreter of the primary literature, using code as its output modality, and a scientific domain schema — gene program, to axis, to composed embedding — as the thing it is filling in. If you have been thinking about coding agents as a way to automate bioinformatics pipelines, this is a step past that. The agent is not automating a pipeline. It is authoring the model.

The empirical claim is that these zero-shot, data-free embeddings hold up. Across mouse and human tissues, they are competitive in biological quality with three baseline classes: conventional data-driven embeddings, foundation-model embeddings, and program-informed methods that use gene-set databases. Batch effects vanish by construction, because the model never touches the counts. Runs are reproducible, because the composition is deterministic given the curated programs. And each dimension is a named, literature-cited gene program. You can audit any axis. You can steer it — the paper shows the composable axes being organized into a developmental tree, which is a natural expression of the compositional design rather than a post-hoc rearrangement.

A few editorial reads. First, this is an interesting counterpoint to the single-cell foundation-model race. Foundation models bet on scale — more cells, more parameters, emergent generalization. This paper bets on the fact that biology already published its priors, and that a coding agent can compile them faithfully. Both bets can be right at the same time; the paper explicitly frames the two as complementary, not competitive. Second, the interpretability story is not a bolt-on. Every axis comes with citations. That is the difference between building interpretation in and adding it later with sparse autoencoders or gene-set enrichment on cluster markers. Third, this is the most concrete instance I have seen of the agent-as-model-author pattern in biology. The template is the guardrail. The LLM's contribution is domain judgment about which programs matter for the scenario and which citations back them up. That is a well-scoped job for a coding agent, and it produces an artifact you can inspect and reuse rather than a chat log.

Two caveats. The preprint is roughly a day old and bioRxiv has not rendered the body yet, so I am reading off a dense abstract — take the specifics with the usual caution and expect the paper's methods and evaluation details to matter when the full text lands. And the whole approach lives or dies on how well the LLM curates gene programs from the literature for a given scenario. In well-mapped tissues, the priors are rich, and this is going to shine. In poorly charted subdomains, hallucinated programs and mis-cited signatures will be the failure mode to watch, and the paper's zero-shot framing means there is no expression-data safety net to catch it. But even with those caveats, this is one of the cleaner ideas I have seen for what a coding agent should actually do in bioinformatics. Not just glue existing tools together. Write the model.
