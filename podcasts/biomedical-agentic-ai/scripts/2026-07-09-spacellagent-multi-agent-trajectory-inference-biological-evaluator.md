# SpaCellAgent — a self-evolving multi-agent LLM for single-cell and spatial trajectory analysis, with a dedicated biological evaluator loop

Paper link: https://arxiv.org/abs/2607.07467

## Script

Today's pick is SpaCellAgent, a KDD 2026 paper posted to arXiv yesterday that puts a multi-agent LLM framework around one of the most persistently annoying analyses in computational biology: trajectory inference over single-cell and spatial transcriptomics data.

The setup is familiar to anyone who has run one of these pipelines. You have a dataset — either dissociated single-cell RNA-seq, or something spatially resolved from Slide-seq, Stereo-seq, or Visium — and you want to reconstruct a developmental trajectory. Pseudotime ordering. Branching. Cell-fate decisions. There is not one tool for this. There is Monocle. There is PAGA and PAGA Tree. There is Slingshot. There is Diffusion Pseudotime. There is Scorpius. There is Slingshot again if it did not work. Each one is grounded in a different mathematical framework — graph abstraction, principal curves, diffusion maps, minimum spanning trees. Each one has inconsistent performance across data topologies. And the community's own benchmarking paper, the Saelens comparison in Nature Biotechnology, essentially concluded that there is no single winner and you need to know your data.

Which means, in practice, that trajectory inference today is a task where the answer depends heavily on which tool you pick, which parameters you tune, and whether the biologist reading the resulting plot recognizes that something has gone wrong. It is exactly the kind of workflow that an agent should be able to run for you.

That is what SpaCellAgent tries to be. Under the hood it is a multi-agent architecture with four roles. A planner takes the user's natural-language query plus a structured metadata profile of the dataset — expression counts, sparsity, spatial modality — and decomposes it into an executable JSON action plan. An executor has two sub-components: a tool selector that maps each planned step to the right method from a registry of single-cell and spatial-aware algorithms, and a coder that writes and runs the actual Python or R script. And then an evaluator — this is the piece that makes the paper interesting — has two sub-components of its own. A code evaluator that catches runtime errors and hands them back to the coder for self-correction. And a biological evaluator that ingests the resulting plots and summaries and uses an LLM to check whether the answer is biologically plausible.

The biological evaluator is the part worth dwelling on. Prior LLM-driven single-cell agents — including CellAgent from last year and, in a broader form, OmniCellAgent that I covered here in early June — have generally treated the LLM as a code-generation and orchestration layer. Errors get caught at the code level. What SpaCellAgent adds is an evaluator whose job is to look at the trajectory that came out and ask questions like: does this rooting make sense given the known cellular hierarchy? Are there any biologically impossible lineage transitions? Is the pseudotime inverted? It has an explicit knowledge-augmented fallback that searches PubMed for tissue-specific markers and analogous cases when the model runs out of stored experience. That is the shift I want to flag. Code correctness is easy for an LLM to verify. Biological correctness — is this an implausible trajectory — is harder, and it is what a domain expert actually spends their time on.

The other piece worth noting is the memory design. There is a local memory that logs the current run's code drafts, execution traces, and error messages, so the coder debugs with context. And a global memory that accumulates verified code snippets, biological metadata, and error-fix pairs across runs. Successful workflows become templates. Common failure modes get proactive remedies. And the tool registry itself can expand — if the agent needs a new package for a spatial algorithm, it can pull it in, and if the analysis using that new tool passes biological validation, the tool gets registered.

The empirical results are on the Saelens 2019 benchmark plus three real spatial datasets — mouse embryonic dorsal midbrain, axolotl neuron regeneration, and an unpublished mouse spinal cord injury dataset. SpaCellAgent matches or beats the classical baselines on trajectory-topology metrics and cuts wall-clock analysis time by about 41 percent relative to a five-expert human baseline. Cross-model sensitivity holds — the framework works with DeepSeek, GPT, and Claude backbones, which suggests the agent design and not the base model is doing most of the work.

Two things I would flag as caveats. First, the private spinal cord dataset is doing real work here — a lot of the "autonomous hypothesis discovery" narrative rests on findings from data the reader cannot verify. Second, the evaluation of the biological evaluator itself is thin. Whether an LLM-based check of "biological plausibility" catches the failure modes that actually matter in a new tissue is an open question, and the paper does not stress-test it head-on.

But the direction is right. Trajectory inference is a nearly ideal test bed for agentic biology: many tools, no principled way to choose among them, and outputs that require biological judgment to verify. SpaCellAgent is not the last word on this problem, but it moves the interesting question one layer up — from whether an agent can run a bioinformatics workflow to whether it can tell that the workflow's answer is biologically wrong.
