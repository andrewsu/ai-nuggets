# SwiftNJ — a correctness-gated coding agent beats a hand-tuned bioinformatics tool

Paper link: https://www.biorxiv.org/content/10.64898/2026.05.28.728410v1

## Script

Today's nugget went up on bioRxiv a couple of days ago from Joe Christensen at Aarhus University, and it is the cleanest empirical answer I have seen this year to the question of whether a current generation coding agent can actually improve on a hand tuned production bioinformatics tool, under what oversight, and by how much. The paper is called SwiftNJ — fast exact neighbour joining via correctness gated coding agents — and the headline result is that a frontier coding agent, operating inside an automated harness with deterministic correctness gates, produces a neighbour joining implementation that runs faster than the long standing reference on fifty eight out of fifty nine benchmark matrices, with a geometric mean runtime ratio of about zero point five six against the comparison baseline.

Let me back up and put the result in context, because the framing matters as much as the number.

Neighbour joining is a distance based method for inferring evolutionary trees, used at very large scale in microbial epidemiology, comparative genomics, and sequence clustering. It is one of those algorithms where the asymptotic complexity has not changed in decades, but the constant factor — how fast a single iteration of the core loop actually runs — has been the subject of years of hand engineering. RapidNJ is the widely cited representative of that hand engineered class. It is a careful native implementation, tuned over time, and it has been the unofficial speed ceiling for exact neighbour joining for about fifteen years. When someone says "the agent beat the bioinformatics tool", you want to know whether the comparison baseline was a polished native implementation or a research script. Here the baseline is the polished one, locally rebuilt for fairness, and that is the point.

The methodological piece worth pulling out is what the author calls a correctness gated optimisation harness. The coding agent is not trusted to validate its own work. Every proposed implementation is run through deterministic correctness gates calibrated against a QuickTree reference, which is the gold standard exact implementation. The harness will accept a faster implementation only if its tree outputs match QuickTree on the calibration inputs. The agent gets to propose, the harness gets to disqualify. That separation is the part that makes this not just another AI-writes-code anecdote. It is an empirical study of how much performance a coding agent can extract from a well defined problem when there is an external truth oracle bounded its solution space.

The numbers. SwiftNJ achieves a geometric mean runtime ratio of zero point five six five against a locally rebuilt RapidNJ native binary across fifty nine distance matrices. Sub-parity — meaning the agent's implementation runs faster — on fifty eight of those fifty nine. That is not a one matrix sweet spot; that is consistent improvement across a benchmark. And on a separate correctness stress test of four hundred shuffled inputs drawn from sixteen small matrices, SwiftNJ matched the QuickTree reference at Robinson-Foulds distance zero. Same trees out, every time, just faster. The combination of consistent speedup and exact correctness preservation is the strong form of the claim.

Three things matter about this beyond the specific bioinformatics result.

First, neighbour joining is exactly the kind of problem where you would expect a coding agent to do well. It is small in code volume, has crisp correctness criteria, runs as a self contained CPU bound loop, and has a long literature of incremental optimisation tricks that a competent coding agent has read during training. If the agent could not beat RapidNJ on this kind of problem, the case for coding agents in performance critical scientific software would be quite weak. That the agent does beat it, by close to a factor of two on geometric mean, sets a meaningful floor for what to expect on neighbouring problems. The right read is not "agents are great at everything" — it is "for tightly scoped, oracle-checkable optimisation work, agents are now competitive with the best hand tuned implementations."

Second, the harness design is the actually transferable contribution. Correctness gates calibrated against an existing reference implementation are a general purpose pattern for putting coding agents to work on scientific software. The bottleneck is rarely the agent's raw capability; the bottleneck is the trust budget the harness gives it. A reference implementation plus a correctness gate plus a fitness function — runtime, memory, accuracy — gives the agent a well defined optimisation surface to climb, and gives the human reviewer a defensible audit trail. That pattern applies straight away to sequence alignment, to read mappers, to variant callers, to phylogenetic inference, to anything in the bioinformatics stack where there is a slow correct reference and a faster production implementation worth challenging. The next round of papers from this corner of the field will probably differ on what the correctness gate is rather than on whether the agent can write code.

Third, the institutional honesty in the paper is unusual and worth noting. The author is explicit that this is a study of one well bounded domain — phylogenetics — and that further work is needed to establish how broadly the harness guided optimisation approach generalises. That is the right framing. The wins so far in coding agents on scientific software have tended to come from problems with this exact shape — crisp objective, available reference, narrow scope. The open question is what happens when any of those go away. Noisier objectives, no reference implementation, broader scope — that is where the trust budget question gets sharp, and that is the regime the rest of the field is now in the early stages of working out.

A few honest read throughs. The fifty nine matrix benchmark is not huge by modern benchmarking standards, and the corpus is a fixed set rather than something pulled from a contemporary epidemiology workflow. The runtime comparison is against a locally rebuilt RapidNJ rather than against the absolute best community optimised binary, which the author acknowledges. And there is no claim about memory footprint or about how the agent's implementation behaves at the very large matrix sizes where neighbour joining is most operationally painful — the small matrix correctness test is the four hundred shuffled inputs at up to two thousand sequences, and the runtime ratio is the geometric mean across the broader corpus, but the constant factor at, say, fifty thousand sequence inputs is not in this paper.

Even with all of that, the structural finding is real. A current generation coding agent, with a well designed correctness gate around it, beats the hand tuned native baseline for exact neighbour joining on a fifty nine matrix corpus by close to a factor of two in geometric mean runtime, with provably identical tree topology on the correctness battery. That is the kind of result that, replicated across a few more tools in the bioinformatics stack, would change how the next generation of performance critical scientific software gets built.
