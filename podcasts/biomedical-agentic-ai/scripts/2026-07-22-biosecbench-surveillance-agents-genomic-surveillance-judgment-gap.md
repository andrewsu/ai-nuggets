# BioSecBench-Surveillance — agents run the right genomics pipeline, then make the wrong calls around it

Paper link: https://arxiv.org/abs/2607.19262

## Script

Today's paper asks a pointed, useful question about agentic AI in biology: can an AI agent actually do the analysis that a genomic-surveillance analyst does? It's called BioSecBench-Surveillance, out of the bioinformatics platform company LatchBio together with the biosecurity screening company Aclid, posted this week.

Start with why it needed doing. Pathogen genomic surveillance — sequencing clinical, wastewater, agricultural, and environmental samples to spot outbreaks, resolve transmission, and catch emerging threats — has a bottleneck, and the bottleneck has moved. Sequencing is cheap and getting cheaper, so the volume of data keeps climbing. But that data only helps once someone analyzes it, and the analysis is expert-driven and doesn't scale. So the obvious pitch is: point an agent at the raw reads and let it figure out the pipeline. This paper tests whether that pitch survives contact with reality.

The benchmark is a hundred evaluations, and the design is the clever part. Each one is a snapshot of a real surveillance workflow, frozen at the moment right before a decision. The agent gets exactly what a human analyst would get — the raw or near-raw sequencing data, some reference files, the sample context — and crucially, nothing about which analysis to run. It has to decide everything: what's the right pipeline, which tools, which databases, which reference genomes, which thresholds. Then it returns a structured answer that's graded deterministically against a validated gold-standard workflow, and every evaluation is peer-reviewed by domain experts before it goes in. The tasks span seven categories — taxonomic classification, variant detection, antimicrobial resistance, source tracking, toxin and virulence, genetic-engineering detection, and open-ended anomaly detection — across six sample types and both short- and long-read sequencing. Each agent works in a sandbox preloaded with the standard open-source bioinformatics stack — assemblers, aligners, classifiers, variant callers — with internet access and a six-hour clock. The same tools a human would reach for.

The headline result: nobody's ready. Across sixteen model-and-harness pairs and nearly four thousand gradable attempts, pass rates ran from about fourteen percent up to about fifty, averaging forty-one. The two best configurations — Anthropic's Opus 4.8 and OpenAI's GPT five-point-five — tied at just barely over fifty percent. Coin-flip territory, on tasks a working analyst finishes comfortably.

But the number isn't the interesting part — the failure mode is. When agents got it wrong, they usually ran the right workflow. The mistakes were in the choices around it: the wrong reference, the wrong threshold, the wrong filter, the wrong normalization. That should ring true to anyone who does bioinformatics for a living. Running the aligner was never the hard part. The hard part is the hundred small judgment calls that don't show up in any tool's help text — the ones you make from experience about this organism, this assay, this sampling context. The agents can drive the tools. They can't yet exercise the judgment that makes the output mean something. And that's a far more useful diagnosis than "agents are bad at biology," because it tells you exactly where the gap is.

A couple of texture points worth flagging. Difficulty tracked the task and the sequencing technology much more than the sample type. Anomaly detection — flag anything unexpected, with no prescribed target — was the worst by a mile at twenty percent, which makes sense: it's open-ended, there's no obvious workflow to pattern-match onto. And long-read data was much harder than short-read, twenty-six percent versus forty-one — a quiet reminder that a lot of an agent's implicit prior is short-read Illumina, and the world isn't only that.

The other finding I'd underline is about harnesses. The authors treat the model and the scaffold — Claude Code, or Codex, or an agent framework they call PI — as independent axes, because the same model behaves wildly differently depending on the wrapper. Refusals are the clearest case: the same OpenAI model declined twenty-seven to twenty-nine percent of tasks under one harness, and only eight or nine percent of the exact same tasks under another. Same weights, same tasks, a threefold difference in willingness to even attempt. If you evaluate a model without pinning the harness, you're not really measuring the model. That's a methodological point the whole agent-evaluation field keeps rediscovering, and it's nice to see it made this cleanly on a biomedical task.

Two caveats on framing. This is a dual-use domain, and the authors handle it responsibly — a public subset with example tasks is on GitHub, but the full evaluation set is held under restricted access, consistent with biosecurity practice. And this is one entry in a series from the same group; they've built comparable execution-based benchmarks for epigenomics and for preclinical pharmacology. The through-line is a philosophy I like: don't quiz an agent on trivia, hand it a realistic workflow snapshot and grade whether it made the right call.

So the takeaway for anyone building agents for bioinformatics: the tool-calling is basically solved, and it was never the bottleneck. The bottleneck is encoded analyst judgment — reference selection, threshold-setting, knowing when a signal is real versus background. Fifty percent, right now, is the number to beat. That's the episode.
