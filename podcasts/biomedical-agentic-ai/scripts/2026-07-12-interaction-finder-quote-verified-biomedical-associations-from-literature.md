# Interaction-finder — LLM-guided literature mining for biological entity associations, with a quoted-passage receipt behind every claim

Paper link: https://www.biorxiv.org/content/10.64898/2026.07.07.736901v1

## Script

Today's pick is Interaction-finder, a bioRxiv preprint posted July tenth from Timo Lassmann's group at The Kids Research Institute Australia in Perth. It is a small, MIT-licensed tool that automates a job every biomedical curator has done by hand: assemble a list of entity-to-entity associations from the literature for a topic where no curated database yet exists.

Setup. Biomedical knowledge graphs live and die by their curation pipelines. The high-quality ones — UniProt, ChEMBL, Reactome, OpenTargets, MONDO — are built by humans reading papers and extracting relationships. That labor bottleneck is why the long tail of interesting questions never gets a database. Ligand–receptor pairs for a niche cell state, cell-type-to-marker mappings for a rare tissue, disease–gene associations behind a poorly characterized syndrome — the graph either does not exist or exists as a spreadsheet in someone's shared drive. LLMs have obvious appeal here, but the honest failure mode of a single-shot prompt is that the model confidently emits associations without a verifiable source, and the honest failure mode of an off-the-shelf deep-research agent is that it summarizes rather than exhaustively enumerates. This paper is the version of that idea done carefully.

Mechanism. The pipeline has three stages. First, keywords: a coding agent reads review articles on the topic and mines them for bridging terms that expand the query beyond the user's initial framing. Second, search: those keywords go against a literature backend — PubMed by default, with Perplexica and OpenAI web search as alternates — in an iterative loop that revises based on what came back. Third, extract: full-text articles are read for candidate associations of the user-specified entity types, and every candidate is scored, quoted, and pinned to a passage in the source text. The final artifact is a self-contained interactive HTML report where each entity pair carries a trust badge, the source passage with the mention highlighted, the evidence quote highlighted, and a written rationale for the quality assessment. If you cannot verify the quote, the tool did not extract it.

Evaluation. Sixty topics across three domains — cell-type to cell-marker, disease to gene, ligand to receptor. Baselines are single-shot prompting and an off-the-shelf deep-research framework. Interaction-finder recalls one-point-two to four-point-three times as many known associations as either baseline, and every extracted quote is verified against source text. On the harder question of what to do with the candidates the tool surfaces that are not in the gold-standard database — the ones you would either dismiss as hallucinations or celebrate as discoveries — the authors ran an independent LLM judge, blind to the extraction reasoning, scoring both gold-standard associations and unverified candidates on the same scale. Across the three domains, unverified candidates score similarly to gold-standard associations. Gold-standard associations are enriched at the top of the ranked list, with overall recall at twenty of zero-point-six-one. That is the number that decides how you use this tool: the top of the ranking is where the known truth concentrates, and the middle is a mix of misses in the gold standard and plausible-but-unverified extensions.

Editorial reads. First, the significance is what quote-level provenance changes about the workflow. Most LLM extraction papers you read this year report an F1 or a precision-recall curve against a gold standard and call it a day. What a curator actually wants from a machine is not a triple, it is a triple plus a receipt — the exact passage that says it, so the human can decide in seconds whether to promote it to the graph. Interaction-finder treats the receipt as first-class. That is the design lesson worth carrying forward even if you never use this specific tool: your extraction interface should return a triple and a passage, and the passage should be verified verbatim, not paraphrased. This is a small architectural commitment with an outsized effect on trust.

Second, the recall gap over a general deep-research framework is the interesting number. The bespoke pipeline more than triples recall in the best case against an off-the-shelf agent given the same topic. That tells you the domain-tuned scaffolding — a keyword-expansion stage that reads reviews, a PubMed-first search backend, a pair-judge sub-agent for extraction — is doing real work. Bare general-purpose agents keep leaving biomedical recall on the table because they do not know the shape of the source landscape. The generalist deep-research pattern is not the ceiling for biomedical curation. It is the floor.

Third, the finding that unverified candidates score similarly to gold-standard associations under a blind LLM judge is a warning shot to anyone treating the current KG as ground truth. It suggests that the space of plausible-but-uncurated associations is large, and a well-tuned LLM extractor mostly surfaces things that a human reader would agree belong in the graph. For a curation pipeline, that reframes the tool's output not as candidates to filter, but as candidates to promote after human review.

Two caveats. This is a fresh preprint, thirty-six hours old, and bioRxiv has not rendered the body yet. This summary is off the abstract and the GitHub README, which is enough to describe the architecture and the headline results but not the ablations that would tell you which component earns the recall gain. And the tool ships with OpenAI as the default extraction backend; the pair-judge is configurable, but the reproducibility posture depends on which model you can access. Overall — a small, well-scoped, MIT-licensed contribution to the boring-but-important layer of biomedical AI, and the receipt-first design is worth stealing.
